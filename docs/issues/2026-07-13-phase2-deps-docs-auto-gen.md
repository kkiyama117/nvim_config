## Implementation notes

### Markdown table pipe escaping

`render_reference.ts` escapes `|` characters in cell content (`\|`) to prevent table corruption. No current TOML entry contains `|`, but the guard is there for robustness.

### `rtp` field rendering

`rtp = ''` (empty string, meaning the plugin directory is not added to runtimepath) is rendered as `""` to distinguish from "field absent" (empty cell). This applies to `dpp.vim` and `dpp-ext-lazy` which both have `rtp = ''` in `deps/dpp.toml`.

### `has_hooks` column

`✓` if any of `hook_add`/`hook_source`/`lua_source` is present; empty otherwise. Hook content is omitted from the table (it is code, not docs). Current state:
- `dpp-ext-toml`: ✓ (`hook_add`)
- `vim-denops/denops.vim`: ✓ (`hook_source`)
- `tree-sitter-manager.nvim`: ✓ (`lua_source` + `hook_source`)
- All other plugins: empty

### `docs/README.md` index line insertion

The `ensureDocsIndexLine()` function in `gen_deps.ts` checks if `references/deps-list.md` already appears in `docs/README.md`; if absent, it trims trailing whitespace from the file and appends a `## Generated references` section with a single bullet linking to the reference doc. The insertion is idempotent: once the link exists, subsequent runs skip the append.
