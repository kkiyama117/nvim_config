## Deviation from design ┬¦5.1 (deno.json location)

**Design ┬¦5.1** specifies the module layout with `scripts/deno.json` (inside the `scripts/` directory). **Plan ┬¦Phase 1 step 1** quoted the same. However, this placement is **non-functional**: Deno's config discovery walks **up** from cwd, not down into subdirectories. The spec ([09-dev-workflow.md](../specifications/09-dev-workflow.md) ┬¦6) and the pre-commit hook (design ┬¦6) both invoke `deno task gen` from the **repo root**, which cannot find `scripts/deno.json`.

Empirical verification (in a scratch directory):

```
$ mkdir -p /tmp/deno-test/scripts && cat > /tmp/deno-test/scripts/deno.json <<'JSON'
{ "tasks": { "gen": "echo found-from-scripts" }, "imports": {} }
JSON
$ cd /tmp/deno-test && deno task gen
error: deno task couldn't find deno.json(c) or package.json. See https://docs.deno.com/go/config
EXIT=1
$ cat > /tmp/deno-test/deno.json <<'JSON'
{ "tasks": { "gen": "echo found-from-root" }, "imports": {} }
JSON
$ cd /tmp/deno-test && deno task gen
Task gen echo found-from-root
found-from-root
EXIT=0
```

**Decision:** place `deno.json` at the **repo root**, not `scripts/deno.json`. This is a mechanical correction required for the design's `deno task gen` invocation to work as the spec/hook intend. The `imports` map (`@std/toml`) and `tasks.gen` content are unchanged from the design. The root `deno.json` does not conflict with the existing `denops/deno.json` (independent configs; no Deno workspace is declared).

**Existing root file:** there was an untracked `deno.json` in the working tree at the start of this session (per git status snapshot: `?? deno.json`). It was empty (1 line, no content). Overwritten with the generator config. The `scripts/` directory was also pre-existing untracked (per git status: `?? scripts/`) ŌĆö it now contains the 4 TypeScript modules.

**Plan revision suggestion:** update design ┬¦5.1 module layout and plan ┬¦Phase 1 step 1 to show `deno.json` at the repo root, not `scripts/deno.json`. Add a note that Deno's config discovery walks up from cwd, so a repo-root-invoked `deno task gen` requires a root-level `deno.json`.
